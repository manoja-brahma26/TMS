# Three Summary Slides for the Finalist Presentation

## Slide 1: Business Value Proposition

### Title

**Transform Travel from Manual Coordination to Intelligent Enterprise Workflow**

### Core Message

Persistent's current TMS works, but it was built for a pre-AI operating model. Our proposal turns travel into a guided, policy-aware, AI-enabled workflow that reduces effort, improves compliance, shortens approval cycles, and creates end-to-end visibility.

### Suggested Slide Points

- simpler trip creation with AI-assisted request capture
- faster approvals with policy-aware routing and low-risk automation
- better agency coordination and booking visibility
- stronger compliance, auditability, and reporting
- reusable platform foundation beyond travel

## Slide 2: Solution Differentiation

### Title

**Why This Approach Wins**

### Core Message

This is not a generic travel tool proposal. It is a brownfield-ready modernization approach built on Persistent's 3Cs architecture and designed to work with the current ecosystem, not against it.

### Suggested Slide Points

- explicit mapping to `Core`, `Context`, and `Coordination`
- strong travel context model with ontology, graph, memory, and semantics
- governed multi-agent design with deterministic controls
- coexistence with email approvals and agency workflows
- phased rollout that earns trust before increasing automation

## Slide 3: Most Critical Point to Remember

### Title

**This Is the First AI-Native Workflow on a Scalable Enterprise Pattern**

### Core Message

The value of this program is bigger than a better travel process. Travel becomes the first production-grade workflow on a repeatable enterprise platform for AI-enabled orchestration, creating a model Persistent can reuse across other functions.

### Suggested Slide Points

- travel is the starting point, not the endpoint
- architecture is reusable for policy-heavy workflows
- governed AI plus enterprise context creates durable value
- business continuity is preserved while modernization progresses
