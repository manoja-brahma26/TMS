# Commercial Proposal for the AI-First Travel Management System

## 1. Commercial Objective

The commercial model should support a phased brownfield modernization with clear scope control, transparent assumptions, and separation between one-time implementation costs and recurring operating costs.

## 2. Recommended Commercial Structure

### Phase-Based Model

- **Phase 0-1:** fixed-price mobilization and discovery
- **Phase 2-4:** milestone-based implementation pricing
- **Phase 5:** fixed-duration hypercare followed by managed support

This model provides commercial predictability early while retaining flexibility for brownfield complexity discovered during implementation.

## 3. Indicative Effort Estimate

| Workstream | Indicative Effort Range |
|---|---:|
| Discovery and blueprint | 6-8 person-weeks |
| Architecture and security design | 4-6 person-weeks |
| UX and workflow design | 4-6 person-weeks |
| Integration engineering | 12-18 person-weeks |
| Workflow and application build | 12-16 person-weeks |
| Data, ontology, and context implementation | 8-12 person-weeks |
| AI agent and orchestration implementation | 8-12 person-weeks |
| Testing and quality assurance | 6-8 person-weeks |
| Change management and training | 4-6 person-weeks |
| Hypercare and stabilization | 4-6 person-weeks |

## 4. Indicative Timeline

| Phase | Duration |
|---|---|
| Mobilization and discovery | 6-8 weeks |
| Core build and assisted workflows | 8-10 weeks |
| Pilot and agency coordination rollout | 8-12 weeks |
| Parallel run and phased rollout | 10-14 weeks |
| Stabilization and support transition | 4-6 weeks |

Total indicative program duration: **36-50 weeks**, depending on brownfield complexity, integration readiness, and rollout scope.

## 5. Staffing Model

### Core Team

- 1 program manager
- 1 business/process lead
- 1 enterprise architect
- 1 AI or agent architect
- 1 data architect
- 1 security/governance lead
- 2-4 engineers for workflow and application build
- 2-3 integration engineers
- 1-2 data/AI engineers
- 1 QA lead plus testing support
- 1 UX lead or product designer
- 1 change management lead

Staffing can scale up or down by phase based on implementation approach and internal team participation.

## 6. Cost Categories

### One-Time Costs

- discovery and solution blueprint
- architecture and design
- integration development
- workflow and UI development
- data and context model setup
- AI agent implementation
- testing and rollout support
- training and change management

### Recurring Costs

- cloud or hosting infrastructure
- model consumption and token usage
- workflow and orchestration platform licenses
- observability and monitoring tools
- support and managed operations

## 7. Licensing Considerations

Licensing assumptions will depend on platform choices, but the commercial response should clearly separate:

- application or workflow platform licenses
- AI model and runtime costs
- data storage and context platform costs
- observability and support tooling
- any third-party connector or integration platform costs

## 8. Infrastructure Assumptions

- non-production and production environments are required
- enterprise identity integration is available
- existing network and security controls will be extended to new services
- hosting model may be cloud, hybrid, or enterprise-hosted depending on security and operations guidance

## 9. Pricing Assumptions

- legacy system documentation may be incomplete
- some integrations may require adapter or intermediary services
- agency communication interfaces may remain partially manual at first
- policy cleanup or normalization may be needed before full automation
- historical data migration scope should be confirmed during discovery

## 10. Commercial Risks

- hidden legacy dependencies can increase implementation effort
- low-quality source data can increase data and testing effort
- changes in scope during rollout can affect schedule and cost
- licensing decisions may materially change recurring cost profile

## 11. Recommended Commercial Positioning

The strongest bid will avoid underpricing the discovery and integration effort. Brownfield modernization programs fail commercially when complexity is ignored up front. A credible proposal should show disciplined estimates, clear assumptions, and optional expansion paths.
